### 📄 Описание 

> **Fixik** — веб-приложение для управления заявками на ремонт и монтаж. Включает ролевую модель (Администратор, Оператор, Исполнитель), гибкий график дежурств, загрузку фото «до/после» и отслеживание статусов. Построено на FastAPI + SQLModel (бэкенд) и React + TypeScript (фронтенд) с Docker-контейнеризацией.

## ✨ Основные возможности

### 🎫 Управление заявками
- Создание, редактирование и отслеживание заявок на ремонт или монтаж.
- Указание адреса, описания, даты/времени, приоритета и типа работ (система).
- Смена статусов: **Ожидает**, **В работе**, **Завершена**, **Отклонена**.
- Комментарии к выполнению и причины отклонения.
- Загрузка фотографий **«До»** и **«После»** для каждой заявки.

### 📅 График дежурств
- Добавление городов и технического персонала.
- Отметка дней дежурств в календаре (кликом по ячейке).
- Режим «Постоянное дежурство» для автоматической отметки всех дней месяца.
- Хранение ФИО и телефона сотрудника (без привязки к email).

### 👥 Роли пользователей
- **Администратор** — полный доступ к заявкам, дежурствам, системам и пользователям.
- **Оператор** — создание заявок, просмотр и редактирование статусов, работа с графиком дежурств.
- **Исполнитель** — просмотр только назначенных заявок, возможность изменить статус на «В работе» или «Завершена», добавить фото «После»
и комментарий.

### 🎨 Интерфейс
- Адаптивная верстка с поддержкой **темной и светлой темы** (переключатель в шапке).
- Удобные таблицы с фильтрацией и сортировкой.
- Уведомления о действиях через `react-hot-toast`.

### 🛠️ Технические особенности
- Асинхронная работа с БД (SQLite) через SQLModel / SQLAlchemy.
- Валидация данных на бэкенде с помощью Pydantic.
- JWT-аутентификация (токены доступа).
- Docker-контейнеризация (отдельные сервисы для бэкенда, фронтенда и базы данных).
- Готовность к развертыванию на любом VPS или локальной машине.

## 🚀 Быстрый старт (локально с Docker)

Убедитесь, что у вас установлены [Docker](https://www.docker.com/) и Docker Compose.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/morten2457/Fixik.git
   cd Fixik
   ```

2. **Создайте файл `.env` в корне** (опционально, для настройки секретного ключа):
   ```
   SECRET_KEY=ваш_супер_секретный_ключ_не_менее_32_символов
   ```

3. **Запустите сборку и поднимите контейнеры:**
   ```bash
   docker-compose up -d --build
   ```

4. **Доступ к приложению:**
   - Фронтенд: http://localhost:5173
   - Бэкенд API: http://localhost:8000/docs (автодокументация Swagger)

5. **Первый вход:**
   При первом запуске автоматически создаётся администратор:
   - Логин: `admin`
   - Пароль: `admin123`

## 🔄 Разработка без Docker (для отладки)

### Бэкенд
```bash
cd backend
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### Фронтенд
```bash
cd frontend
npm install
npm run dev
```
